# 系统架构

## 整体架构

```
                      ┌──────────────────────┐
                      │    Vue 3 前端 (5173)  │
                      │  异动看板 | 管理 | RAG  │
                      └─────────┬────────────┘
                                │ REST + WebSocket
                                │ nginx 反向代理
                      ┌─────────▼────────────┐
                      │   FastAPI (uvicorn)   │
                      │   localhost:8000      │
                      ├──────────────────────┤
                      │  API 路由层            │
                      │  stocks/signals/      │
                      │  limit-up/bili/       │
                      │  rag/ws/monitor       │
                      ├──────────────────────┤
                      │  TaskManager          │
                      │  (生命周期+自动重启)    │
                      ├──────────────────────┤
                      │  后台监控任务 (async)   │
                      │  股票监控 | B站监控     │
                      │  涨停抓取              │
                      ├──────────────────────┤
                      │  SignalQueue          │
                      │  asyncio.Queue        │
                      │  consumer → MySQL     │
                      │    → 钉钉推送          │
                      └────┬──────┬──────┬───┘
                           │      │      │
                    ┌──────▼──┐ ┌─▼──┐ ┌▼────────┐
                    │  MySQL  │ │JSON│ │ ChromaDB │
                    │ 6张表   │ │缓存│ │ 向量库   │
                    └─────────┘ └────┘ └──────────┘
                           │
                    ┌──────▼──────┐
                    │  钉钉机器人  │
                    │ 推送 + 回调  │
                    └─────────────┘
```

## 数据流

### 股票盯盘
```
东方财富 API ──→ stock_core（行情/信号检测）
                     │
                     ├──→ 钉钉推送（实时告警）
                     ├──→ MySQL（signals 表）
                     ├──→ WebSocket（前端实时）
                     └──→ ChromaDB（LLM 分析入库）
```

### 涨停分析
```
东方财富 API ──→ limit_up（收盘后抓取涨停榜）
                     │
                     ├── 过滤（主板≥9.8%/创业板≥19.5%）
                     ├── 获取概念板块
                     ├── LLM 分析涨停原因
                     ├──→ MySQL（limit_up_daily 表）
                     ├──→ 钉钉推送（涨停速报）
                     └──→ ChromaDB（原因分析入库）
```

### B站监控
```
B站 API ──→ bili（动态轮询）
                  │
                  ├──→ LLM 摘要
                  ├──→ 钉钉推送
                  ├──→ MySQL（bili_dynamics 表）
                  └──→ WebSocket
```

## 技术栈

| 层 | 技术 | 选型理由 |
|----|------|---------|
| 后端框架 | FastAPI | 异步原生、自动 OpenAPI、社区活跃 |
| 数据库 | MySQL | 阿杰熟悉、商业级、有扩展空间 |
| 向量库 | ChromaDB | 零配置、文件级、够用 |
| LLM | DeepSeek API | 已有 Key、便宜、中文好 |
| Embedding | DeepSeek text-embedding-v2 | 走 API 不占本地资源 |
| 前端框架 | Vue 3 + Vite | 轻量、现代、好上手 |
| 实时通信 | WebSocket | 信号秒级推送 |
| 部署 | uvicorn + nginx | 简单稳定 |

## 设计原则

1. **前后端分离** — 后端只提供 API，前端只负责展示
2. **异步化** — 所有 I/O 操作用 async/await
3. **不重复造轮子** — 行情展示用东方财富，复盘分析用自己系统
4. **JSON 文件不动** — 现有缓存/凭据继续使用，不迁移
5. **不用 ORM** — 直接原生 SQL，保持可控
