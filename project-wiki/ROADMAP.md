# 开发路线图

## Phase 1: 后端底座（≈6.5 天）

**目标：** FastAPI 能跑、MySQL 能写、API 能调

| 步骤 | 内容 | 天数 | 状态 |
|------|------|------|------|
| 1 | 项目骨架 + 依赖安装 | 0.5 | ✅ 已完成 |
| 2 | 数据库初始化（建表 + 连接池） | 0.5 | ✅ 已完成 |
| 3 | 信号入库 + B站动态入库（监控→队列→MySQL） | 1.5 | ✅ 已完成 |
| 4 | WebSocket 实时推送 | 0.5 | 📝 |
| 5 | FastAPI 启动 + 后台任务管理 | 1 | ✅ 已完成 |
| 6 | API 路由实现（20+ 端点） | 2 | 📝 |
| 7 | 钉钉回调迁移 + 兼容旧入口 | 0.5 | 📝 |

**产出：**
- 后端 API 可调用、监控正常运行
- 信号/B站动态写入 MySQL
- WebSocket 实时推送可用
- 原项目 `stock` 仍可并行运行
